<details>
<summary><b>⬡ &nbsp;go_http_checker — Production HTTP Endpoint Monitor</b></summary>
<br>

A lightweight but production-ready HTTP health checking utility written in Go. Monitors endpoint availability, response time, and status codes — built for infrastructure operators who need a fast, no-dependency tool they can deploy anywhere.

| Attribute | Detail |
|---|---|
| **Stack** | Go, net/http, concurrent workers, structured output |
| **Purpose** | HTTP endpoint monitoring and uptime verification for infrastructure ops |
| **Performance** | Concurrent checks across multiple endpoints; minimal memory footprint |
| **Security** | TLS verification, configurable timeout and retry policies |
| **Impact** | Operational visibility for self-hosted infrastructure without heavy tooling overhead |
| **Repository** | [![GitHub](https://img.shields.io/badge/parothegreat/go_http_checker-1E1B4B?style=flat-square&logo=github&logoColor=white)](https://github.com/parothegreat/go_http_checker) |

Designed to be a single-binary tool deployable on any Linux server. The checker runs concurrent HTTP requests using Go's goroutine pool, collects status codes and response times, and reports failures immediately. Ideal for quick infrastructure audits, post-deploy smoke tests, and lightweight uptime monitoring without spinning up a full observability stack.

<br>
</details>

<details>
<summary><b>⬡ &nbsp;rfid-access-control-system — Institutional Physical Access Platform</b></summary>
<br>

A full RFID-based physical access control system built for SMK Mitra Industri MM2100. Manages student and staff access to controlled areas via RFID card authentication, with a Go backend handling card validation and access logging in real time.

| Attribute | Detail |
|---|---|
| **Stack** | Go, RFID hardware integration, database backend, REST API |
| **Scale** | Institutional deployment — school campus with multiple access points |
| **Performance** | Sub-200ms card read-to-response latency for seamless entry flow |
| **Security** | Card UID validation, access log audit trail, role-based area permissions |
| **Impact** | Replaced manual attendance and access tracking with automated, auditable system |
| **Repository** | [![GitHub](https://img.shields.io/badge/teamitmivhs/rfid--access--control--system-1E1B4B?style=flat-square&logo=github&logoColor=white)](https://github.com/teamitmivhs/rfid-access-control-system) |

The system bridges physical RFID hardware with a software backend — card reads are processed by a Go service that validates UIDs against the database, logs timestamp and location, and triggers access decisions in real time. Built as part of the teamitmivhs org, this project demonstrates practical embedded-systems-meets-backend engineering at institutional scale.

<br>
</details>
